## 简介

从上一课来看，负载均衡似乎只在数据中心内执行。但是，需要在全局和局部范围内进行负载平衡。让我们了解一下两者各自的功能：

- **全球服务器负载均衡（GSLB）**：GSLB涉及跨多个地理区域的流量负载分配。
- **本地负载均衡**：这是指在数据中心内实现的负载均衡。这种类型的负载平衡侧重于提高数据中心托管服务器的效率和更好的资源利用率。

让我们分别了解以下两种技术。

## 全局服务器负载均衡

GSLB 确保全球到达的流量负载被智能地转发到数据中心。例如，数据中心的电源或网络故障需要将所有流量重新路由到另一个数据中心。GSLB 根据用户的地理位置、不同位置的托管服务器数量、数据中心的健康状况等因素做出转发决策。

在下一课中，我们还将了解 GSLB 如何提供自动区域故障转移。GSLB 服务可以安装在本地或通过 负载平衡即服务 (LBaaS)获得。

### DNS 中的负载平衡

我们知道 DNS 可以使用多个 IP 地址响应 DNS 查询。[在关于DNS](https://www.educative.io/collection/page/10370001/4941429335392256/5728619204182016)的课程中，我们讨论了可以通过 DNS 进行负载平衡，同时查看`nslookup`. DNS 使用一种简单的技术重新排序 IP 地址列表以响应每个 DNS 查询。因此，不同的用户得到一个重新排序的IP地址列表。它导致用户访问不同的服务器来处理他们的请求。通过这种方式，DNS 将请求的负载分散到不同的数据中心。这是执行 GSLB。特别地，DNS 使用循环法来执行负载平衡，如下所示：

如上所示，DNS 中的循环将客户端以严格的循环顺序转发到数据中心。但是，循环法具有以下限制：

- 不同的 ISP 拥有不同数量的用户。为许多客户提供服务的 ISP 将向其客户提供相同的缓存 IP，导致终端服务器上的负载分布不均匀。
- 因为循环负载平衡算法不考虑任何终端服务器崩溃，它会继续分配崩溃服务器的 IP 地址，直到缓存条目的 TTL 到期。在这种情况下，服务的可用性可能会因 DNS 级负载平衡而受到影响。

尽管有其局限性，循环法仍被许多 DNS 服务提供商广泛使用。此外，DNS 对缓存条目使用短 TTL，以在不同数据中心之间进行有效的负载平衡。

> **注意：** DNS 不是 GSLB 的唯一形式。应用程序交付控制器 (ADC) 和基于云的负载平衡（[稍后讨论](https://www.educative.io/collection/page/10370001/4941429335392256/6596709789990912#Implementation-of-load-balancers)）是执行 GSLB 的更好方法。

什么是应用程序交付控制器 (ADC)？

应用交付控制器 (ADC) 是应用交付网络 (ADN) 的一部分。它们可以被认为是提供各种服务（包括负载均衡）的 LB 的超集。ADC 的主要任务是执行 Web 加速以减少服务器群的负载。第 3 层和第 7 层之间的一些众所周知的服务包括缓存、SSL 卸载、代理/反向代理服务、IP 流量优化等等。ADC 还实现 GSLB。

### 需要本地负载均衡器

DNS 在平衡负载方面起着至关重要的作用，但它受到以下限制：

- 较小的 DNS 数据包（512 字节）不足以包含服务器的所有可能 IP 地址。
- 对客户行为的控制有限。客户端可以从收到的 IP 地址集中任意选择。某些接收到的 IP 地址可能属于繁忙的数据中心。
- 客户端无法确定与其建立连接的最近地址。尽管可以实施 DNS 地理定位和类似任播的解决方案，但它们并不是微不足道的解决方案。
- 如果发生故障，由于缓存机制，通过 DNS 恢复可能会很慢，尤其是当 TTL 值较长时。

为了解决上面的一些问题，我们需要以本地LB的形式再做一层负载均衡。在下一课中，我们将讨论有关本地负载均衡器的不同细节。

## 什么是本地负载均衡？

本地负载平衡器驻留在数据中心内。它们的行为类似于反向代理，并尽最大努力将传入请求分配给可用服务器池。**传入客户端的请求无缝连接到使用虚拟 IP 地址 ( VIP** )的 LB。

DNS 可以被视为全球服务器负载平衡器 (GSLB) 吗？

隐藏答案

是的，实际上有两种方法可以进行全局流量管理（GTM）：**通过 ADC 的 GTM**：一些 ADC 实现 GSLB。在这种情况下，ADC 可以实时查看托管服务器并根据数据中心的健康状况和容量转发请求。**GTM through DNS**：DNS通过分析客户端的IP位置来做GSLB。对于每个请求域名 IP 的用户（例如[www.educative.io](http://www.educative.io/)），基于 DNS 的 GSLB 转发数据中心的 IP 地址，在地理上更接近请求的 IP 位置。
